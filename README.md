# Stellar Window 2.0 — прототип

Инсталляция Stellar Residence: экран — «проём в мир» от первого лица,
картинка параллаксится от положения головы — ощущение, что стоишь внутри. Спека: `docs/superpowers/specs/2026-06-10-stellar-window-splat-design.md` (v1: `2026-06-10-stellar-mirror-prototype-design.md`).

## Запуск

    npm install
    npm run dev        # http://localhost:5173, разрешить камеру
    npm test           # юнит-тесты

### ⚠️ Первый запуск требует интернета

WASM и модели MediaPipe грузятся с CDN (jsdelivr / googleapis), потом кэшируются браузером. На демо без сети прототип не стартует.

### Лучше смотреть в полноэкранном режиме

Миры с реальными фото дают параллакс по карте глубины; gaussian-splat миры — настоящий 3D.

## Клавиши

| Клавиша | Действие |
|---|---|
| `1`–`9` | выбор мира напрямую |
| `W` / `M` | следующий / предыдущий мир |
| `A` | режим выравнивания мира (стрелки, [ ], - =, Shift — крупный шаг) |
| `D` | debug-панель (FPS, задержка, позиция головы) |
| `C` | калибровка (размеры экрана, смещение камеры, FOV вебки) |

Клавиши привязаны к физическим позициям (e.code) — работают в любой раскладке.

## Калибровка

Дефолты сняты с MacBook Pro 14" (30.41 × 19.74 см, камера в чёлке: Y = +9.5 см) —
из коробки честный масштаб. Для другого экрана нажми `C` и введи ширину/высоту
видимой области в сантиметрах (линейкой) и смещение камеры от центра экрана.
Подсказка для Samsung Odyssey G9 49": экран 119.3 × 33.6 см; вебка стоит сверху →
Y ≈ 33.6/2 + 1 ≈ 17.8 см. FOV вебки — 63°, если не знаешь точно.

## Чек-лист приёмки (Stellar Window 2.0)

- [ ] Мир от первого лица, параллакс от движения головы
- [ ] Подход к экрану плавно «вводит» в мир (dolly), отход — выводит, без дрожи на границе
- [ ] Переключение миров `1..9`/`W`/`M` с шторкой < 0.5 с
- [ ] Демо-сплат рендерится (Spark), photo25d-миры рендерятся (спальня, балкон)
- [ ] Мир из другой технологии подключается папкой + meta.json без правки кода
- [ ] Debug: render ≥ 50 fps; потеря лица/два лица/запрет камеры — как в v1
- [ ] `gen-world.mjs` без ключа падает с понятным сообщением; с ключом собирает папку мира

## Замена контента

Миры лежат в `public/assets/worlds/<имя>/` (сплат или 2.5D-фото) — формат описан в `docs/CONTENT_GUIDE.md` §5.

Все пути и габариты — в `src/scenes/config.ts`.

Сейчас миры показывают **реальные фото заказчика** как «2.5D-фото»: к каждому снимку
сгенерирована карта глубины (Depth Anything V2), и шейдер выдвигает ближние объекты
к зрителю — параллакс работает по-настоящему (кровать отделяется от стены, перила
балкона — от города). Gaussian-splat миры (demo-splat) дают полноценный 3D через Spark.

### Как заменить/добавить фото

Одной командой: `node scripts/add-photo.mjs <файл> <имя>` — создаст папку
`public/assets/worlds/<имя>/`, скопирует фото, сгенерирует карту глубины и запишет
`meta.json`. Затем добавь `'<имя>'` в `worlds` в `src/scenes/config.ts`.
Промпты для Gemini и весь процесс — в [docs/CONTENT_GUIDE.md](docs/CONTENT_GUIDE.md).

### Требования к фото (если генерировать через Gemini/Midjourney)

- **Интерьер (спальня):** съёмка с уровня глаз (~150 см), как будто стоишь у стены
  и смотришь в комнату. Ширина от 2000 px. Без людей в кадре.
  Текущие ассеты сделаны так: референс → ре-рендер с уровня глаз в Gemini
  (nano banana) → апскейл Real-ESRGAN ×4 → карта глубины.
- **Вид (балкон/окно):** от 2000 px по ширине, вечер/сумерки смотрятся лучше всего.
  Передний план с перилами/подоконником — плюс (красиво параллаксится).
  Настоящая равнопромежуточная панорама 2:1 тоже подойдёт (вернём сферу).
