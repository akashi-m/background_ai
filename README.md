# Stellar Mirror — прототип

«Виртуальное зеркало» для инсталляции Stellar Residence: видишь себя в 3D-интерьере,
фон параллаксится от положения головы. Спека: `docs/superpowers/specs/2026-06-10-stellar-mirror-prototype-design.md`.

## Запуск

    npm install
    npm run dev        # http://localhost:5173, разрешить камеру
    npm test           # юнит-тесты

### ⚠️ Первый запуск требует интернета

WASM и модели MediaPipe грузятся с CDN (jsdelivr / googleapis), потом кэшируются браузером. На демо без сети прототип не стартует.

### Режим «Окно» — запускать в полноэкранном режиме

Рама окна строится из физических размеров экрана; в неполноэкранном окне браузера внешние бруски рамы остаются за кадром.

## Клавиши

| Клавиша | Действие |
|---|---|
| `M` | режим «Зеркало» (ты в интерьере) |
| `W` | режим «Окно» (вид на город) |
| `D` | debug-панель (FPS, задержка, позиция головы) |
| `C` | калибровка (размеры экрана, смещение камеры, FOV вебки) |

Клавиши привязаны к физическим позициям (e.code) — работают в любой раскладке.

## Калибровка

Дефолты сняты с MacBook Pro 14" (30.41 × 19.74 см, камера в чёлке: Y = +9.5 см) —
из коробки честный масштаб. Для другого экрана нажми `C` и введи ширину/высоту
видимой области в сантиметрах (линейкой) и смещение камеры от центра экрана.
Подсказка для Samsung Odyssey G9 49": экран 119.3 × 33.6 см; вебка стоит сверху →
Y ≈ 33.6/2 + 1 ≈ 17.8 см. FOV вебки — 63°, если не знаешь точно. После смены
размеров экрана перезагрузи страницу (геометрия рамы окна строится на старте).

## Чек-лист приёмки (из спеки)

- [ ] Видишь себя в 3D-интерьере, фон параллаксится от движения головы
- [ ] Debug: render ≥ 50 fps, сегментация ≥ 24 fps, возраст кадра камеры < 50 мс
- [ ] `W`/`M` переключают режимы, шторка < 0.5 с
- [ ] Закрыть камеру рукой → картинка плавно уезжает в нейтральный ракурс, без рывков
- [ ] Два человека в кадре → трекается ближний
- [ ] Запрет камеры → понятное сообщение, не чёрный экран
- [ ] Замена интерьера/панорамы = правка `src/scenes/config.ts`

## Замена контента

Все пути и габариты — в `src/scenes/config.ts`. Появится GLTF-квартира от заказчика —
прописать в `interiorGltfUrl`, фоны отключатся сами.

Сейчас оба режима показывают **реальные фото заказчика** из `images/` как
«2.5D-фото»: к каждому снимку сгенерирована карта глубины (Depth Anything V2),
и шейдер выдвигает ближние объекты к зрителю — параллакс работает по-настоящему
(кровать отделяется от стены, перила балкона — от города).
Запасной вариант — процедурная спальня: `mirrorBackground: 'procedural'` в конфиге.

### Как заменить/добавить фото

1. Положить фото в `public/assets/` (например, `room2.jpg`)
2. Сгенерировать карту глубины: `node scripts/gen-depth.mjs public/assets/room2.jpg public/assets/room2_depth.png`
3. Прописать пути в `src/scenes/config.ts`

### Требования к фото (если генерировать через Gemini/Midjourney)

- **Интерьер (зеркало):** съёмка с уровня глаз (~150 см), как будто стоишь у стены,
  где висит зеркало, и смотришь в комнату. Ширина от 2000 px. Без людей в кадре.
  Текущее фото снято с верхней точки — работает, но ракурс «зеркало висит высоко»;
  фронтальный ракурс с уровня глаз будет убедительнее.
- **Город (окно):** от 2000 px по ширине, вечер/сумерки смотрятся лучше всего.
  Передний план с перилами/подоконником — плюс (красиво параллаксится).
  Настоящая равнопромежуточная панорама 2:1 тоже подойдёт (вернём сферу).
