# Stellar Window 2.0 — first-person на гауссовых сплатах

**Дата:** 2026-06-10
**Статус:** утверждён
**Заменяет:** концепцию «зеркала» из `2026-06-10-stellar-mirror-prototype-design.md` (прототип v1 реализован и остаётся фолбэком). Решение по итогам v1: вырезание фигуры посетителя смотрится чужеродно, 2.5D-фото имеет потолок качества («тянучки» на краях объектов). Новая планка: **идеальная картинка от первого лица**, цена и железо не ограничены.

## 1. Цель

Посетитель подходит к экрану и видит квартиру Stellar Residence **от первого лица**, «как будто сам стоит внутри»:

1. Трекинг лица (позиция головы x/y/z) — без контроллеров и кнопок.
2. Голова влево/вправо/вверх/вниз → честный параллакс (off-axis проекция).
3. Подошёл ближе → камера плавно «въезжает» в комнату до ~1.5 м; отошёл — выехала.
4. Картинка фотореалистична: настоящее 3D без артефактов 2.5D (никаких «резиновых краёв»).
5. Контент-агностичность: из одной картинки сцена **генерируется**; если позже появится
   съёмка реальной квартиры или 3D-модель — интеграция без изменения кода.

## 2. Ключевое решение: 3D Gaussian Splatting как единый формат сцены

Сцена — гауссов сплат (`.spz`/`.ply`): фотореалистичное представление 3D-пространства,
рендерится в браузере через **Spark 2.0** (стриминговый 3DGS-рендерер для three.js от
World Labs, с LOD). Сплат — обычный three.js-объект: существующий рендер-цикл,
off-axis камера и трекинг не меняются.

Пути получения сплата (в порядке появления у заказчика):

| Источник | Инструмент | Когда |
|---|---|---|
| Одна картинка (рендер/фото) | **Marble World API** (image → explorable 3D world → экспорт .spz/.ply, платный план) | сейчас |
| Реальная квартира/шоурум | съёмка телефоном ~10 мин → Postshot / Luma → .spz | когда дадут доступ |
| 3D-модель архвиза | рендеры по орбите → тренировка сплата; либо GLTF напрямую | когда дадут исходники |

Отвергнутые альтернативы:
- *Unreal Engine 5* — требует 3D-модель (её нет), каждая квартира = недели работы художника; «картинка → сцена» не работает. Дверь не закрыта: Marble экспортирует сплаты в UE.
- *Прокачанный 2.5D (LDI + inpainting)* — потолок качества ниже сплатов, «въезд» внутрь ограничен. Текущая 2.5D-реализация остаётся фолбэк-форматом мира.

## 3. Архитектура

```
КОНТЕНТ-ПАЙПЛАЙН (офлайн)              РАНТАЙМ (браузер, киоск)
картинка → Marble API ─┐               вебкамера → MediaPipe → One-Euro
съёмка → Postshot ─────┼→ worlds/<имя>/  → позиция головы (x,y,z, см)
3D-модель → сплат ─────┘  world.spz      → параллакс: off-axis проекция (есть)
                          meta.json      → въезд: dollyFromEyeZ() — НОВОЕ
                                         → сцена: Spark SplatMesh — НОВОЕ
                                         → фейд-переключение миров (есть)
```

Слои независимы: пайплайн производит папки миров; рантайм их потребляет; формат
`meta.json` — контракт между ними.

## 4. Формат мира

`public/assets/worlds/<имя>/`:

```jsonc
// meta.json
{
  "title": "Спальня",
  "format": "splat",            // "splat" | "photo25d" (фолбэк)
  "file": "world.spz",          // для photo25d: photo.png + depth.png
  "transform": {                // выравнивание сгенерённого мира к экрану
    "position": [0, 0, 0],      // см
    "rotationYDeg": 0,
    "scale": 1.0                // сплаты приходят в произвольном масштабе
  },
  "dollyMaxCm": 150,            // насколько глубоко можно «въехать»
  "source": "marble:<job-id>"   // происхождение, для воспроизводимости
}
```

Валидация meta.json — чистая функция с юнит-тестами; битый файл → понятная ошибка
с именем файла, не чёрный экран.

## 5. Движение зрителя

- **Параллакс (x/y):** существующий off-axis конвейер; усиление `parallaxGain`
  масштабируется от ширины экрана (на проде ~120 см → 1:1).
- **Въезд (z):** новая чистая функция `dollyFromEyeZ(eyeZcm)` → проезд камеры в см:
  - зона покоя: z ≥ 80 см → проезд 0;
  - 80 → 30 см → плавный проезд 0 → `dollyMaxCm` (smoothstep);
  - гистерезис ±5 см против дрожи на границе зоны;
  - сглаживание тем же One-Euro подходом.
  - Реализация: сцена сдвигается к плоскости экрана (`scene.position.z += dolly`),
    проекционная математика не трогается.

## 6. Выравнивание мира (alignment)

Сгенерённые миры приходят в произвольном масштабе и ориентации. В debug-панели —
режим выравнивания: ручки сдвига/поворота/масштаба сцены, значения пишутся в
localStorage и печатаются в консоль готовым JSON для вставки в meta.json.
Выровнял один раз — мир готов для всех посетителей.

## 7. Контент-пайплайн (скрипты)

- `scripts/gen-world.mjs --image <фото> --name <имя>`: Marble World API
  (ключ `WORLDLABS_API_KEY` в `.env`) — создать генерацию → дождаться → скачать
  `.spz` → собрать папку мира с meta.json. Понятные ошибки на каждом шаге.
- Существующие `gen-depth.mjs` / `add-photo.mjs` остаются для фолбэка photo25d.
- Документация оператора в `docs/CONTENT_GUIDE.md` дополняется разделом про миры.

## 8. Переключение миров

Миров сколько угодно (спальня, гостиная, балкон…). Список — в `src/scenes/config.ts`.
Клавиши: `1..9` — прямой выбор мира, `W`/`M` сохраняют смысл «следующий/предыдущий».
Переключение — существующая стейт-машина с фейдом (< 0.5 с). Оба соседних мира
держатся в памяти; остальные подгружаются лениво (Spark стримит с LOD).

## 9. Железо

- **Прототип:** MacBook M4 (Spark с LOD тянет в браузере).
- **Прод:** Windows-ПК с RTX 4080+, дисплей 4K 85–98" вертикально, штатная вебкамера
  (MediaPipe доказал себя в v1). Киоск-режим Chrome. Запас по fps — кратный.

## 10. Ошибки

- Сплат/meta не загрузились → существующий оверлей с именем файла.
- Marble API: ошибки квоты/генерации → внятные сообщения в скрипте, job-id для повтора.
- Трекинг: затухание при потере лица, ближайшее лицо, клампы eye.z — уже решено в v1.
- Низкий fps на конкретном железе: Spark LOD + ручка качества (sh-degree/количество
  частиц при экспорте) — фиксируется на этапе выравнивания мира.

## 11. Тестирование

- **Юниты:** `dollyFromEyeZ` (зона покоя, ход, гистерезис), валидация meta.json,
  все 33 существующих теста остаются зелёными.
- **Ручной «тест магии v2»:** параллакс + въезд телом; fps ≥ 50 на M4 по debug-панели;
  переключение миров < 0.5 с; качество сплата субъективно «как фото» в пузыре
  движения (±50 см параллакс, 150 см въезд).

## 12. Критерии приёмки

1. Из одной картинки командой `gen-world.mjs` собирается папка мира, мир открывается
   в приложении и выглядит фотореалистично в пузыре движения.
2. Подход к экрану плавно «вводит» в комнату на dollyMaxCm; отход — выводит; на границе
   зоны нет дрожи.
3. Параллакс и въезд работают одновременно и не ломают друг друга.
4. Мир из другого источника (например, скачанный готовый сплат-пример) подключается
   только папкой + meta.json, без правки кода.
5. fps ≥ 50 на M4, переключение миров < 0.5 с, все юниты зелёные.

## 13. Вне скоупа

- Сегментация посетителя (showPerson) — выключена, код остаётся в репо.
- Процедурная спальня — остаётся фолбэком, из активной конфигурации убирается.
- Полноценная прогулка по квартире (вариант c) — следующая итерация.
- Режим ожидания / датчики приближения / LED — продакшн-этап инсталляции.
