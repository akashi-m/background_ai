# Stellar Mirror Lux — мастер-спека

**Дата:** 2026-06-11
**Статус:** утверждён (мастер-дизайн; каждый подпроект получает свою спеку/план)
**Заменяет:** v1 (зеркало-прототип) и v2 (first-person сплаты) как продакшн-цель.
v2 остаётся рабочим прототипом и донором кода.

## 1. Видение

Люксовая интерактивная инсталляция Stellar Residence для ТЦ (референс-рендер
заказчика: чёрный портал с подсветкой, зеркало в полный рост, боковая панель
с видом города, нижний тач-столик с планировками и формой заявки).

Сценарий: посетитель подходит → заставка-слайдшоу плавно сменяется «зеркалом» —
посетитель видит себя в полный рост внутри фотореалистичного интерьера Stellar
Residence → тач-кнопками выбирает один из четырёх интерьеров (Light / Modern /
Classic / Ferré) и вид из окна → на столике выбирает планировку и оставляет
заявку (имя + телефон) → QR ведёт на сайт.

Планка качества — **люкс, без компромиссов MVP**. Бюджет железа не ограничен.

## 2. Уроки v1/v2 (почему так)

- v1: сегментация MediaPipe поверх обычной вебки давала «дышащие» края и
  чужеродную фигуру — главный провал качества. → Решение: depth-камера
  (надёжность) + нейроматтинг (красота) + гармонизация (правдоподобие).
- v2: параллакс от головы — рабочая магия, но для зеркального сценария
  вторична; сохраняем как опциональный эффект фона (есть в рендерере).
- Генеративный контент-конвейер (Gemini → глубина → мир) — работает, остаётся
  основным способом производства интерьеров до получения рендеров заказчика.

## 3. Архитектура

Два процесса на одной Windows-машине (RTX 5090) + контент + операционка.

```
CAPTURE-СЕРВИС (Python)                РЕНДЕРЕР+UI (веб, Chrome-киоск, наш код v2)
┌──────────────────────────────┐       ┌──────────────────────────────────────┐
│ SourceAbstraction:           │       │ Интерьеры: бэкплейты 4–8K + глубина  │
│  zed (прод) | webcam | file  │       │  (+опц. параллакс из v2)             │
│ Маска: depth-gate (ZED)      │ RGBA  │ PersonLayer: приём потока, шейдер:   │
│  → нейроматтинг 4K           │──────▶│  альфа-рекомбинация, LUT интерьера,  │
│  (RVM ONNX → TensorRT)       │WebRTC │  light wrap, контактная тень, зерно, │
│ NVENC, side-by-side RGB|A    │       │  зеркальный флип                     │
│ WS-телеметрия: presence,     │──────▶│ Стейт-машина: IDLE → APPROACH →      │
│  distanceCm, joints, health  │  WS   │  MIRROR → (timeout) → IDLE           │
└──────────────────────────────┘       │ UI-слой: тач-кнопки интерьеров,      │
                                       │  смена вида из окна, QR, слайдшоу    │
                                       └──────────────────────────────────────┘
ТАЧ-СТОЛИК — второе окно киоска (route /table): планировки, форма лидов;
  синхронизация с зеркалом по WebSocket (выбранный интерьер ↔ подсветка планировок)
ЛИДЫ — локальная очередь (SQLite) → Telegram-бот + CSV-экспорт; офлайн-стойкая
ADMIN — route /admin: калибровка, перезагрузка контента, health, аналитика
```

### 3.1 Контракты между процессами (фиксируются и тестируются)

- **Видеопоток**: WebRTC (localhost), H.264 NVENC, 60 fps, кадр = side-by-side
  упаковка `[RGB | A]` двойной ширины; рендерер распаковывает в шейдере.
  Бюджет задержки стекло-в-стекло: ≤ 100 мс.
- **Телеметрия**: WebSocket, JSON-сообщения
  `{type: 'presence', present, distanceCm, personCount}`,
  `{type: 'joints', ...}` (для будущих жестов),
  `{type: 'health', fps, gpuLoad, model}`. Частота 15 Гц.
- Рендерер живёт без capture-сервиса: нет потока → остаётся в IDLE
  (никогда не показывает чёрный экран или сломанный композит).

### 3.2 Dev/prod-паритет (ключ к «идеальному»)

Capture-сервис обязан работать в трёх режимах источника:
`--source zed` (прод), `--source webcam` (дев на Маке, маттинг MediaPipe/RVM-CPU),
`--source file:путь` (детерминированные тесты). Весь пайплайн и рендерер
разрабатываются и тестируются на MacBook M4 без спец-железа; ZED-путь
активируется на проде. Контракты идентичны во всех режимах.

## 4. Качество композита — три слоя

1. **Надёжность (физика)**: ZED 2i neural depth → маска по порогу расстояния
   (зона посетителя 0.8–2.5 м), толпа ТЦ за зоной отсекается; выбор ближайшего
   человека при нескольких в зоне.
2. **Красота (нейроматтинг)**: depth-маска как prior → RVM (реалтайм, 4K) с
   путём апгрейда на MatAnyone 2; альфа без фликера, волосы, тонкие края.
3. **Правдоподобие (гармонизация в шейдере рендерера)**:
   - цветовой LUT, извлечённый из бэкплейта интерьера (автоген в контент-пайплайне);
   - light wrap: фон подмешивается в 2–4 px контура фигуры;
   - контактная тень: мягкий градиент под ногами (позиция ног из joints);
   - зерно/шум, согласованный с бэкплейтом; лёгкий bloom поверх композита.
   - **Физика будки**: тёплый ключевой свет слева сверху (как в интерьерах) —
     требование к конструкции, прописывается в монтажную документацию.

## 5. Стейт-машина опыта

```
IDLE ──(presence, дистанция < 250 см)──▶ APPROACH ──(1.2 с анимация)──▶ MIRROR
 ▲                                                                        │
 └──────────(absence > 10 с, fade 1 с)──────────────────────────────────┘
```

- **IDLE**: слайдшоу люкс-рендеров (кроссфейд 8 с) + логотип + «подойдите ближе»;
- **APPROACH**: зеркало «проявляется» (анимация яркости + частицы);
- **MIRROR**: композит + UI; выбранный интерьер сохраняется на сессию;
- Все тайминги/пороги — в конфиге, меняются без пересборки.

## 6. UI

- **Зеркало (основной экран, портрет)**: верх — логотип + «выберите интерьер»,
  4 тач-кнопки стилей (Light/Modern/Classic/Ferré), низ — «сменить вид из окна»,
  QR-код. Типографика и палитра по референс-рендеру (чёрный + тёплое золото).
  Тач — IR-рамка поверх экрана либо встроенный PCAP тотема.
- **Столик (второй дисплей 32", route /table)**: выбор планировки (2/3/4 спальни),
  форма «Имя + Телефон → Отправить», мини-превью выбранного интерьера.
- Мультиязычность: ru/uz/en, переключатель на обоих экранах, тексты в JSON.

## 7. Лиды и аналитика

- Заявка: валидация телефона (+998…), запись в SQLite немедленно,
  отправка в Telegram-бот заказчика; при офлайне — очередь с ретраями;
  CSV-экспорт из /admin. Никаких внешних SaaS без согласования.
- Аналитика (локально, агрегаты в /admin): сессии, длительность, выборы
  интерьеров, конверсия в заявку. Без сохранения видео/фото посетителей
  (приватность: кадры живут только в видеопамяти).

## 8. Контент

4 интерьера × (бэкплейт ≥4K портрет + карта глубины + LUT + вид из окна).
Производство: наш конвейер (Gemini-генерация по референсам заказчика →
глубина → LUT-экстракция скриптом). Когда заказчик даст архвиз-рендеры —
подключаются той же папкой. Слайды IDLE — из тех же бэкплейтов.
Формат папки наследует worlds/ из v2 с расширением meta.json
(`lut`, `windowViews[]`, `slideshow`).

## 10. Подпроекты (порядок реализации)

1. **capture-service**: абстракция источников, маттинг, WebRTC+WS выход,
   dev-режим Mac. Приёмка: эталонное видео → RGBA-поток в браузере ≥30 fps на M4.
2. **renderer-композит**: приём потока, шейдер гармонизации (LUT, light wrap,
   тень, зерно), стейт-машина IDLE/APPROACH/MIRROR. Приёмка: «тест люкса» —
   композит на эталонном видео неотличим «на расстоянии беглого взгляда».
3. **Lux UI + лиды + столик + admin**: две страницы киоска, Telegram, аналитика.
4. **Контент**: 4 интерьера + LUT-экстрактор + слайды + виды из окон.
5. **Ops**: Windows-киоск, сервисы, watchdog, автостарт, удалённый доступ,
   дневной ребут, мониторинг, прогон 12 ч.

Каждый подпроект проходит полный цикл: спека → план → subagent-driven
реализация с двойным ревью.

## 11. Стандарты «идеального» софта (для всех подпроектов)

- TDD для всей чистой логики; типизация строгая (TS strict / Python typed+mypy).
- **Golden-тесты**: эталонный ролик прогоняется через пайплайн, ключевые кадры
  сравниваются с эталонами (допуск PSNR); запуск в CI на каждый коммит.
- Chaos-приёмка: kill capture-сервиса → IDLE ≤ 1 с; kill рендерера → автоперезапуск
  ≤ 5 с; отключение камеры/сети — деградация без чёрных экранов.
- Soak: 12-часовой прогон с записью fps/память/GPU; утечки = блокер.
- Конфигурация — файлы, не код; контент — папки, не пересборка.
- Логи структурные, ротация; все ошибки видимы в /admin.

## 12. Критерии приёмки (вся система)

1. «Тест люкса»: посетитель в зеркале выглядит частью интерьера (края волос,
   свет, тень) — подтверждается на живой демке заказчику.
2. Полный сценарий: подход → проявление → выбор 4 интерьеров → смена вида →
   заявка на столике → лид в Telegram. Без клавиатуры/мыши.
3. Задержка зеркала ≤ 100 мс, рендер 60 fps на проде / ≥30 на dev-Маке.
4. 12 ч непрерывной работы без деградации; восстановление после любого сбоя
   без участия человека.
5. Замена контента — папкой, без пересборки; калибровка — из /admin.
6. Лиды не теряются при офлайне; аналитика считает сессии и конверсию.

## 13. Вне скоупа мастер-спеки

- Жесты как управление (joints собираем — UI-жесты в следующей итерации).
- Видеозапись/шаринг роликов посетителем (потенциальный этап 2: QR со своим фото).
- Интеграция CRM заказчика (пока Telegram + CSV).
