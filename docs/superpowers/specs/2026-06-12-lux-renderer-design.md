# Lux-рендерер (подпроект №2 Stellar Mirror Lux) — спека

**Дата:** 2026-06-12
**Статус:** утверждён
**Родитель:** `2026-06-11-stellar-mirror-lux-design.md` (§3, §4, §5)
**Контракт входа:** `capture/CONTRACT.md` (заморожен 2026-06-12) — единственный
источник истины о потоке и телеметрии. Рендерер не делает предположений сверх него.

## 1. Цель

Веб-рендерер принимает RGBA-поток фигуры посетителя и телеметрию от
capture-сервиса и компонует фигуру в фотореалистичный интерьер с гармонизацией
(LUT, light wrap, контактная тень, зерно), управляя опытом через стейт-машину
IDLE → APPROACH → MIRROR. Результат на dev-Маке: живой «люкс-композит»
от вебки через оба процесса.

## 2. Ключевое решение: интерьеры = существующие миры

Рендерер строится ПОВЕРХ кодовой базы Stellar Window 2.0 (`src/`), а не вместо:

- интерьер = мир `photo25d` (бэкплейт + глубина) — за спиной посетителя
  интерьер живёт лёгким параллаксом от головы (магия v2 сохраняется);
- `WorldSwitcher` переключает интерьеры (4 стиля в подпроекте №4;
  пока bedroom/balcony как плейсхолдеры);
- трекинг головы v2 (HeadTracker) продолжает давать параллакс фона;
  presence для стейт-машины берётся ИЗ ТЕЛЕМЕТРИИ capture (один источник истины
  о присутствии, спека §3.1 мастера).

## 3. Новые модули (`src/lux/`)

| `experience.ts` | Стейт-машина IDLE/APPROACH/MIRROR; входы: present, distanceCm, streamStatus; тайминги/пороги из конфига; выход: фаза + прогресс 0..1 переходов | TDD: все переходы, протухание, гистерезис дистанции |
| `sbsLayer.ts` | Фуллскрин-слой фигуры: распаковка SBS (RGB слева, альфа = .r справа), зеркальный флип, feather краёв | шейдер — ручная проверка + юнит на геометрию UV-преобразования |

| `shadow.ts` | Контактная тень: позиция/размер эллипса из bbox телеметрии (низ bbox = ноги), сглаживание One-Euro-подходом; рендер — мягкий радиальный градиент между миром и фигурой | TDD: математика bbox→эллипс, поведение при bbox=null |

## 4. Конвейер кадра (изменение рендер-цикла)

```
1. мир (фон с параллаксом) → WebGLRenderTarget        ← было «сразу на экран»
2. RT → экран (фуллскрин-блит)
3. контактная тень (если MIRROR и bbox есть)
4. слой фигуры: SBS-распаковка → LUT → light wrap(размытый RT) → зерно → feather
5. фейды переходов фаз (чёрная шторка WorldSwitcher остаётся для смены миров)
```

Light wrap использует уменьшенную (1/4) размытую копию RT фона — отдельный
дешёвый блюр-пасс. Весь композит — на существующем three-рендерере.

## 5. Стейт-машина опыта

```
IDLE ──(present && distanceCm < approachCm(250))──▶ APPROACH ──(1.2 c)──▶ MIRROR
 ▲                                                                          │
 ├──(absence > exitSec(10), fade 1 c)◀──────────────────────────────────────┘
 └──(streamStatus != live || телеметрия протухла > staleSec(2)) — немедленно
```

- В IDLE слой фигуры и тень выключены полностью (не рендерятся).
- APPROACH: кроссфейд слайдшоу → мир + проявление фигуры (альфа 0→1).
- Сломанный композит не показывается никогда: любой сбой → IDLE.
- Все тайминги/пороги — `src/lux/config.ts`, меняются без пересборки кода
  сцены (числа в одном месте).

## 6. Расширение формата мира

`meta.json` мира получает опциональные поля:
- `lut: "interior.cube"` — LUT гармонизации для этого интерьера;
- `shadowStrength: 0..1` (дефолт 0.5).
Без полей — identity-LUT и дефолтная тень: существующие миры валидны без правок.

## 7. Dev-режим и дебаг

- Полный e2e на Маке: `uv run capture --source webcam|file:клип` +
  `npm run dev` → живой композит.
- URL capture-сервиса — в конфиге (`http://localhost:8765`).
- Дебаг-клавиши (расширение существующей панели): `F1..F4` — вкл/выкл
  LUT / light wrap / тень / зерно; `F5` — форс-фаза (IDLE→APPROACH→MIRROR
  по кругу, для разработки без телеметрии); статус потока в debug-панели.

## 8. Ошибки

| Сбой | Поведение |
|---|---|
| capture не запущен / упал | статус `down`, IDLE, реконнект с backoff, баннер в debug-панели |
| телеметрия молчит > 2 с | статус `stale`, IDLE |
| битое сообщение телеметрии | игнор + счётчик в debug, поток не рвём |
| .cube битый/отсутствует | identity-LUT + console.warn, мир работает |
| capture перезапустился | автореконнект, MIRROR возвращается сам |

## 9. Тестирование

- TDD: телеметрия-парсер, experience, .cube-парсер, математика тени,
  backoff (фейк-таймеры). Существующие 48 веб-тестов остаются зелёными.
- Ручной «тест люкса»: на живом композите F1–F4 поштучно — каждый слой
  гармонизации виден и улучшает картинку; края волос без бахромы на
  тёмном и светлом интерьере.
- Хаос вручную: kill capture → IDLE ≤ 1 с; старт capture → MIRROR ≤ 5 с.

## 10. Критерии приёмки

1. e2e на Маке: вебка → capture → рендерер → «я в интерьере» с гармонизацией.
2. Фазы переключаются живой телеметрией (подошёл/ушёл) и форсятся F5.
3. F1–F4 переключают слои гармонизации на лету.
4. Kill/restart capture: IDLE ≤ 1 с, автовозврат ≤ 5 с, ни одного сломанного кадра.
5. Рендер 60 fps на M4 при живом потоке (по debug-панели).
6. Все юниты зелёные; существующие тесты не сломаны.

## 11. Вне скоупа

Тач-UI, лиды, столик, мультиязычность (№3); реальные 4 интерьера, LUT-экстрактор,
слайды (№4); ZED/NVENC/киоск (Ops); выпиливание HeadTracker-параллакса (остаётся).
