# Lux UI + лиды + столик + admin (подпроект №3 Stellar Mirror Lux) — спека

**Дата:** 2026-06-12 (этапирована 2026-06-13)
**Статус:** утверждён; исполняется ЭТАПАМИ по решению заказчика:
- **Этап 1 (сейчас): только зеркало, качественно** — §3 (UI зеркала) +
  дев-самопроверка UI (Playwright/Preview) + полировка по живому e2e.
  Без QR (уходит в этап 2 вместе с «всяким»).
- **Этап 2 (потом):** столик (§4), лиды (§5), аналитика+admin (§6),
  i18n (§7), QR (§8).
**Родитель:** `2026-06-11-stellar-mirror-lux-design.md` (§6, §7)
**Опирается на:** Lux-рендерер (§2: фазы/mirrorOpacity из `Experience`),
capture-сервис (aiohttp-приложение — хост для лидов/админки).

## 1. Цель

Превратить технический композит в продукт: люксовый тач-UI на зеркале
(4 интерьера, QR, мультиязычность), тач-столик с планировками и формой заявки,
офлайн-стойкие лиды в Telegram, /admin с health и аналитикой.

## 2. Архитектурные решения

1. **Бэкенд лидов = изолированный модуль capture-сервиса** (`capture/src/capture/leads/`):
   aiohttp уже поднят и вечно жив; на киоске остаётся 2 процесса. Жёсткая изоляция:
   своя SQLite, свои тесты, ни одного импорта из matting/pipeline; выключается
   флагом `--no-leads` (dev-режимы рендерера без лидов работают).
2. **Синхронизация зеркало ↔ столик = BroadcastChannel**: оба окна в одном Chrome
   (два дисплея одного ПК) — нулевая инфраструктура, типизированный протокол.
3. **UI = HTML-оверлей** поверх канваса: люксовая типографика на CSS делается
   на порядок быстрее и точнее шейдеров; палитра/шрифты — CSS-переменные
   в одном файле (брендбук заказчика вставится заменой переменных).

## 3. UI зеркала (`src/lux/ui.ts`, слой в `index.html`, `src/lux/lux-theme.css`)

- **MIRROR**: верх — логотип + «Выберите интерьер»; 4 тач-кнопки стилей
  (подписи из meta.json миров, маппинг стиль→мир в `LUX_CONFIG.interiors`);
  низ — «Сменить вид из окна» + QR. Оверлей появляется/гаснет синхронно
  с `mirrorOpacity` (CSS opacity из того же значения).
- **IDLE**: логотип + «Подойдите ближе» поверх слайдшоу; кнопки скрыты.
- Палитра по референс-рендеру: чёрный фон, тёплое золото (#d9b878±),
  тонкие линии; шрифты — системная замена до получения брендбука.
- Активная кнопка подсвечена; нажатие → `WorldSwitcher.switchTo` + событие
  аналитики + BroadcastChannel.
- «Сменить вид из окна»: циклит `windowViews[]` из meta.json мира
  (новое опциональное поле: список файлов видов; без поля — кнопка скрыта).
  Смена вида = горячая подмена текстуры окна… В photo25d-мирах окно — часть
  бэкплейта, поэтому вид из окна = ОТДЕЛЬНЫЙ мир-вариант: `windowViews[]`
  содержит имена миров-вариантов (bedroom-night и т.п.), кнопка = switchTo.
- Тач: обычные click; никакой клавиатуры в проде (F/буквенные клавиши остаются
  дев-инструментами).

## 4. Тач-столик (`table.html`, `src/lux/table.ts`)

Второе окно Chrome на втором дисплее (route `/table.html` того же Vite/статики):
- выбор планировки: карточки 2/3/4 спальни (картинки `public/assets/plans/`,
  плейсхолдеры до материалов заказчика);
- мини-превью выбранного интерьера (синхронно с зеркалом);
- форма: Имя + Телефон (+998 маска) + кнопка «Отправить» → POST /leads;
  состояния: отправка → «Спасибо! Менеджер свяжется» (10 с) → сброс;
- ошибка сети → «Попробуйте ещё раз», данные формы не теряются.

**Протокол BroadcastChannel `stellar-lux`** (typed, TDD):
```ts
{ type: 'interior', index: number }          // зеркало → столик
{ type: 'lead-submitted' }                   // столик → зеркало («Спасибо» на зеркале)
{ type: 'lang', lang: 'ru' | 'uz' | 'en' }   // любое окно → все
```

## 5. Лиды (`capture/src/capture/leads/`)

- `POST /leads` `{name, phone, interior, plan, lang}`:
  валидация (имя непустое ≤100; телефон нормализуется к +998XXXXXXXXX,
  иначе 400 с полем ошибки) → INSERT в SQLite (`leads.db`, status=pending) →
  202 немедленно (Telegram не блокирует ответ).
- Фоновый воркер: pending-лиды → Telegram Bot API (`TELEGRAM_BOT_TOKEN`,
  `TELEGRAM_CHAT_ID` из .env) → status=sent; ошибки → retry c backoff
  (1м → 5м → 15м, бесконечно), status=pending остаётся.
- Без токена в .env: лиды копятся в SQLite, воркер спит, /admin предупреждает.
- `GET /leads.csv` — экспорт (имя, телефон, интерьер, планировка, язык, время, статус).
- TDD: валидация/нормализация телефона, машина статусов воркера
  (фейк-Telegram: успех/ошибка/таймаут), идемпотентность ретраев.

## 6. Аналитика + /admin

- Рендерер шлёт события: `POST /events` `{type: 'session_start'|'session_end'|
  'interior_selected'|'lead_submitted', ...payload}` → SQLite (`events`).
  session_start = вход в MIRROR, session_end = возврат в IDLE (длительность).
  Отправка fire-and-forget, недоступность бэка не мешает рендеру.
- `GET /admin` (HTML в capture): health pipeline (fps/errors), статус Telegram,
  таблица последних лидов + CSV-кнопка, агрегаты: сессии/день, средняя
  длительность, выборы интерьеров, конверсия в заявку.
- Сервер слушает ТОЛЬКО 127.0.0.1 (уже так); удалённый доступ — Ops (туннель).

## 7. Мультиязычность (`src/lux/i18n.ts`, `public/locales/{ru,uz,en}.json`)

- `t(key)` c фолбэком ru при отсутствии ключа (console.warn);
- переключатель «RU / UZ / EN» на зеркале и столике (синхронно через канал);
- сброс на ru при возврате в IDLE (новый посетитель);
- TDD: фолбэк, отсутствующий ключ, смена языка.

## 8. QR

`scripts/gen-qr.mjs --url <адрес> --out public/assets/qr.png` (библиотека
`qrcode` в devDependencies). URL в `LUX_CONFIG.qrUrl`; до финального URL —
плейсхолдер сайта заказчика.

## 9. Ошибки

| Сбой | Поведение |
|---|---|
| /leads недоступен со столика | «Попробуйте ещё раз», форма не очищается |
| Telegram недоступен | лид в SQLite pending, ретраи, /admin предупреждает |
| Битый locale-ключ | фолбэк ru + warn |
| BroadcastChannel недоступен | окна работают автономно (выбор не синхронится, заявки работают) |
| /events недоступен | рендер не страдает, события теряются молча (аналитика ≠ критична) |

## 11. Критерии приёмки

1. Полный сценарий на Маке: зеркало + столик в двух окнах → выбор интерьера
   синхронится в обе стороны → заявка с валидацией → лид в SQLite →
   (с токеном) в Telegram → виден в /admin, конверсия посчитана.
2. UI «проявляется» вместе с зеркалом, в IDLE — только логотип и приглашение.
3. Языки переключаются на обоих экранах синхронно; сброс в IDLE.
4. Офлайн-Telegram: лид не теряется, ретраится, /admin предупреждает.
5. Все юниты зелёные (web + python); существующие 75+32 не сломаны.

## 12. Вне скоупа

Реальные планировки/брендбук/QR-URL (вставка по мере получения), 4 реальных
интерьера (№4), удалённый /admin (Ops), печать или CRM-интеграция лидов.
