# Зерно на весь кадр — спека

**Дата:** 2026-06-13
**Статус:** утверждён в чате («ну го»)
**Родитель:** Lux-рендерер §4. После цвет-матча — следующий слой «спаивания»
фигуры и фона в одну фотографию.

## 1. Проблема

Сейчас зерно добавляется внутри шейдера фигуры (`rgb += (hash-0.5)*uGrain`) —
только на фигуру. Фон-интерьер зерна не имеет → разное зерно у слоёв
*разделяет* их (фигура «шумит», комната гладкая = «наклейка»). Плёночное
зерно должно лежать на всём кадре одинаково, объединяя слои.

## 2. Решение: финальный фуллскрин-пасс

Перестроить конец рендер-цикла: слои (блит фона, слайдшоу, тень, фигура)
собираются в offscreen `compositeRT`, затем один финальный `grainMat`-пасс
накладывает шум на весь кадр `compositeRT → экран`.

- **Зерно убирается из шейдера фигуры** (uGrain/uGrainOn/hash оттуда уходят).
- Новый `compositeRT` (полный размер); слои рисуются в него, не на экран.
- `grainMat` (GLSL1, как blit): `c = texture(tSrc,vUv); if(on) c += (hash-0.5)*uGrain`.
- Тогл F4 (`toggles.grain`) теперь правит финальный пасс; сила/анимация (uTime)
  те же. Шторка смены миров (fade) — после зерна, как и была, поверх.
- Применяется всегда, когда композит на экране (и в MIRROR, и в IDLE-слайдшоу) —
  единый плёночный слой; гладкий слайд тоже получает «киноплёнку».

## 3. Риск

Перестройка целевых RT в `render()`: следить за clear (compositeRT чистится
раз перед слоями; экран покрывается непрозрачным grain-блитом). Фидбэк-луп
исключить: compositeRT ≠ sceneRT (блит читает sceneRT, пишет в compositeRT).

## 4. Тестирование

Шейдер/таргеты — визуальная приёмка (прецедент компонента: рендер юнитами
не покрыт). Существующие web-тесты (87) зелёные, typecheck чист. Приёмка:
live на интерьере — зерно видно и на фигуре, И на стене комнаты одинаково;
F4 выключает на всём кадре; на крупном плане текстура шума единая через
границу фигуры (нет «шумная фигура / гладкий фон»).
