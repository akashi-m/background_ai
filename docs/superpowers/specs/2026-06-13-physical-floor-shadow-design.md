# Физическая тень от ламп (пол + мебель + стены) — спека

**Дата:** 2026-06-13
**Статус:** утверждён в чате
**Родитель:** Lux-рендерер (`2026-06-12-lux-renderer-design.md` §4),
заменяет силуэтную тень `groundShadowMat` (коммит 38bd4b7) на физически
рассчитанную.

## 1. Цель

Тень фигуры на полу должна вычисляться по реальной геометрии комнаты (позиции
ламп, поза и положение посетителя), а не быть экранной имитацией «вниз». Это
убирает ощущение «за мной картинка»: тень меняет направление и длину при
смещении посетителя и повторяет позу (рука/наклон/нога).

## 2. Объём

**В скоупе:** тень падает на ВЕСЬ интерьер — пол, мебель, стены — через точную
карту глубины комнаты из Blender (Z-pass) как приёмник тени. От 2-3 главных ламп,
поза-зависима (из живого силуэта), направление/длина/мягкость физичны (высота
лампы + рост + позиция). Сглаживание против дрожи.

**Вне скоупа (→ вариант B, бэклог):** ОТБРАСЫВАТЕЛЬ остаётся плоским billboard-
силуэтом (плоскость в точке `F` лицом к камере) — тень на мебели верной формы и
места, но «от плоского тебя», без истинного объёма тела и самозатенения. Объём —
3D-прокси тела (вариант B).

## 3. Данные из Blender (разово, точные)

**Карта глубины комнаты:** Blender Z-pass → `living/roomdepth.exr` (или 16-бит
PNG с min/max в lights.json) — точная дистанция камера→каждый пиксель (пол,
мебель, стены). Приёмник тени. Рендерится той же камерой плейта, разово.

`public/assets/worlds/living/lights.json` (генерится скриптом
`scripts/export-lights.mjs` или вручную из MCP):
```json
{
  "lamps": [
    { "name": "Key_Living_Warm", "pos": [4.5, 3.2, 2.3], "weight": 1.0 },
    { "name": "Spot_LV_1",       "pos": [6.4, 1.5, 2.8], "weight": 0.6 },
    { "name": "Ceiling_LED",     "pos": [4.0, 1.5, 3.0], "weight": 0.4 }
  ],
  "camera": { "pos": [1.35,2.2,1.62], "target": [8.0,1.5,1.05], "lens": 18,
              "sensorFit": "vertical", "resX": 1080, "resY": 1920 },
  "floorZ": 0.0
}
```
Мир в координатах Blender (X восток, Y север, Z вверх; см. layout-память).
`weight` — относительная плотность тени от лампы (сумма нормируется).

## 4. Геометрия (чистые функции, `src/lux/shadowGeom.ts` + шейдер)

Механика — shadow-mapping через буфер глубины: приёмник = весь интерьер
(roomdepth), отбрасыватель = силуэт-билборд. Чистые функции для матриц/позиции
тестируются юнитами; сам per-pixel тест — в шейдере (визуальная приёмка).

**4.1 Позиция/рост посетителя → мир.** `personFloorWorld(telemetry, cfg)`:
- `F` (точка на полу под центром фигуры): по `distanceCm` вдоль оси взгляда +
  гориз. смещение из bbox-центра и FOV камеры → мировая точка на `floorZ`;
- `H_person` (м): из высоты bbox и дистанции через FOV; клампим [1.4,2.0], деф 1.7.

**4.2 Билборд отбрасывателя.** Вертикальная плоскость в `F`, нормаль — к камере,
высота `[floorZ, floorZ+H_person]`, ширина из аспекта силуэта. Функция
`billboardUV(worldPointOnPlane, F, H)` → UV силуэта для сэмпла альфы.

**4.3 Камера и реконструкция.** `cameraMatrices(camera)` (Blender-плейт-камера,
НЕ живой off-axis) → view+proj. `worldFromDepth(screenUV, depth, camInv)` →
мировая 3D-точка пикселя комнаты. `projectToScreen(world, viewProj)` + та же
cover-fit-трансформация, что у backplate (`coverMat` uUvScale) — иначе тень
разъедется с картинкой. Юнит-тесты на известных точках (сверка с Blender).

**4.4 Тест тени (в шейдере, на лампу).** Для мировой точки пикселя `Pw`:
луч `Pw → L`; пересечение с билборд-плоскостью; `billboardUV` → сэмпл альфы
силуэта; пиксель в тени, если альфа непрозрачна И пересечение между `Pw` и `L`
(окклюдер впереди). Мягкость — несколько сэмплов (PCF) / блюр; затухание с
дистанцией от контакта.

## 5. Рендер (`src/lux/compositor.ts`)

Заменяет `groundShadowMat` на полноэкранный **shadow-пасс по глубине** — между
блитом фона и фигурой:
- вход: `compositeRT` (фон), `roomdepth`, силуэт (person SBS), `lights.json`
  (лампы+камера), `F`,`H_person` (сглажены), cover-fit uUvScale;
- для каждого пикселя: `worldFromDepth` → для каждой из 2-3 ламп тест §4.4 →
  суммарный коэффициент затенения `s` (с `weight` лампы, PCF-мягкостью);
- **умножаю фон на `(1 − shadowStrength · s · mirrorOpacity)`** → тень ложится
  на пол/мебель/стены по их реальной глубине;
- фигура рисуется ПОВЕРХ (её область фон-тень не трогает — там она сама).

Параметры в `LUX_CONFIG.shadow` (strength, softness, bias против акне) — без
пересборки. Bias по глубине — чтобы поверхность не затеняла сама себя.

## 6. Сглаживание

`F`, `H_person` сглаживаются One-Euro/экспонентой (как `SmoothedShadow` сейчас),
чтобы дрожание bbox/дистанции не дёргало угол. Потеря фигуры → плавное затухание.

## 7. Ошибки / деградация

| Сбой | Поведение |
|---|---|
| нет `lights.json`/`roomdepth` у мира | фолбэк: текущая силуэт-тень «вниз» (groundShadowMat), без падения |
| телеметрия без bbox/distance | фиксированная точка стояния (центр лаунжа) |
| `H_person` шумит | клампим + сглаживаем; дефолт 1.7 |
| лампа ниже роста (`z ≤ H`) | пропускаем эту лампу (нефиз.) |
| shadow-acne (самозатенение) | depth-bias в тесте §4.4 |

## 8. Тестирование (TDD, hard rules — приоритет)

Юниты (`shadowGeom.test.ts`):
- `personFloorWorld`: дальше дистанция → `F` дальше по оси взгляда; смещение
  bbox влево → `F` смещается верно; рост из bbox+дистанции в диапазоне [1.4,2.0].
- `cameraMatrices`/`projectToScreen`: известная мировая точка пола → известный
  экранный пиксель (сверка с Blender-рендером прокси).
- `worldFromDepth`: пиксель + глубина + camInv → мировая точка (round-trip с
  projectToScreen = тождество).
- пересечение луч↔билборд + `billboardUV`: луч от точки к лампе сквозь силуэт →
  верная UV; мимо силуэта → нет тени.
Шейдер/много-ламп/заползание на мебель — визуальная приёмка против Blender-
прокси «Human 2» (эталон): тень на полу И на диване/стене совпадает по форме,
направлению, длине, мягкости.

## 9. Этапы (декомпозиция)

1. Экспорт `lights.json` (2-3 лампы + камера) и `roomdepth` (Z-pass) из Blender.
2. `shadowGeom.ts` (позиция/рост, матрицы камеры, реконструкция из глубины,
   пересечение с билбордом) + юниты — ядро без рендера.
3. Шейдер: ОДНА лампа, проверить на живом, что тень на ПОЛУ ложится как у прокси.
4. Включить заползание на мебель/стены (через roomdepth), 2-3 лампы, PCF-софт,
   сглаживание, depth-bias.
5. Приёмка против прокси (пол + диван + стена), тюнинг `LUX_CONFIG.shadow`.

## 10. Критерии приёмки

1. Сдвиг посетителя вбок/по глубине меняет направление и длину тени физично.
2. Поза (рука/наклон/нога) отражается в тени.
3. Тень корректно **заползает на диван/стену** по их реальной глубине, совпадая
   с Blender-рендером прокси «Human 2» в той же точке (форма, направление, длина).
4. Нет дрожи и shadow-acne; потеря фигуры → плавное затухание;
   без `lights.json`/`roomdepth` — фолбэк на старую силуэт-тень.
5. Веб-юниты зелёные (87 не сломаны); рендер держит текущий fps (60 на M4).
