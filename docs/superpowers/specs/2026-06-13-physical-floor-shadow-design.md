# Физическая контактная тень на полу — спека

**Дата:** 2026-06-13
**Статус:** утверждён в чате
**Родитель:** Lux-рендерер (`2026-06-12-lux-renderer-design.md` §4),
заменяет силуэтную тень `groundShadowMat` (коммит 38bd4b7) на физически
рассчитанную.

## 1. Цель

Тень фигуры на полу должна вычисляться по реальной геометрии комнаты (позиции
ламп, поза и положение посетителя), а не быть экранной имитацией «вниз». Это
убирает ощущение «за мной картинка»: тень меняет направление и длину при
смещении посетителя и повторяет позу (рука/наклон/нога).

## 2. Объём

**В скоупе:** тень на ПОЛУ — физическая проекция силуэта от 2-3 главных ламп
(направление, длина, мягкость из высоты лампы + роста + позиции). Поза-зависима
(берётся из живого силуэта). Сглаживание против дрожи.

**Вне скоупа (→ вариант B, бэклог):** тень, заползающая на стены/мебель;
истинная объёмная тень (нужен 3D-прокси тела + реалтайм-сцена); межобъектное
затенение. Billboard-допущение: силуэт — плоскость, стоящая в точке `F` лицом
к камере.

## 3. Данные из Blender (разово, точные)

`public/assets/worlds/living/lights.json` (генерится скриптом
`scripts/export-lights.mjs` или вручную из MCP):
```json
{
  "lamps": [
    { "name": "Key_Living_Warm", "pos": [4.5, 3.2, 2.3], "weight": 1.0 },
    { "name": "Spot_LV_1",       "pos": [6.4, 1.5, 2.8], "weight": 0.6 },
    { "name": "Ceiling_LED",     "pos": [4.0, 1.5, 3.0], "weight": 0.4 }
  ],
  "camera": { "pos": [1.35,2.2,1.62], "target": [8.0,1.5,1.05], "lens": 18,
              "sensorFit": "vertical", "resX": 1080, "resY": 1920 },
  "floorZ": 0.0
}
```
Мир в координатах Blender (X восток, Y север, Z вверх; см. layout-память).
`weight` — относительная плотность тени от лампы (сумма нормируется).

## 4. Геометрия (чистые функции, `src/lux/shadowGeom.ts`)

Все в мировых метрах Blender, затем проекция на экран матрицей камеры.

**4.1 Позиция/рост посетителя → пол.** `personFloorWorld(telemetry, cfg)`:
- `F` (точка на полу под центром фигуры): по `distanceCm` (дистанция от камеры
  вдоль оси взгляда) и горизонтальному смещению из bbox-центра + FOV камеры →
  мировая точка на `floorZ`.
- `H_person` (рост, м): из высоты bbox (доля кадра) и дистанции через FOV;
  клампим в [1.4, 2.0], дефолт 1.7 при шуме.

**4.2 Тень одной лампы.** `floorShadow(F, H_person, lamp, floorZ)`:
- горизонт. вектор от лампы к фигуре `dir = normalize((F − lamp.pos).xy)`;
- гориз. дистанция `d = |(F − lamp.pos).xy|`;
- длина тени `Lsh = H_person · d / max(lamp.pos.z − H_person, ε)`
  (низкая лампа → длинная; высокая → короткая);
- «дальняя» мировая точка тени (куда падает голова): `tip = F + dir · Lsh`.
- Возвращает `{ baseWorld: F, tipWorld: tip, dir, len: Lsh }`.

**4.3 World→screen.** `projectFloor(world, camera)`: матрица вида+проекции
камеры **Blender-плейта** (НЕ живой off-axis-камеры — фон рисуется этой
камерой) по lens+sensorFit+aspect → нормированные координаты **изображения
плейта** [0..1]. Затем применяется ТА ЖЕ cover-fit-трансформация, что у
backplate (compositor `coverMat` uUvScale), → финальные экранные [0..1], иначе
тень не совпадёт с полом на картинке. Чистая, юнит-тест на известных точках.

Каждая лампа даёт экранную трансформацию силуэта: ступни силуэта → `F` на экране,
макушка → `tip` на экране. Это аффинно-перспективный сдвиг/скос силуэта вдоль
направления тени с масштабом `len/H_person`.

## 5. Рендер (`src/lux/compositor.ts`)

Заменяет одиночный `groundShadowMat`-пасс:
- для каждой из 2-3 ламп — пасс проекции силуэта (тот же sampleA, что и тень
  сейчас, но UV-трансформ из §4.3 вместо простого `uDrop`): чёрный, блюр
  (мягкость растёт с длиной тени), прозрачность `shadowStrength · weight ·
  mirrorOpacity`;
- пассы суммируются в `compositeRT` ПЕРЕД фигурой (как сейчас → закрыто фигурой
  везде кроме пола).
- Смягчение: чем дальше от ступней (по длине), тем мягче и прозрачнее (penumbra).

Параметры в `LUX_CONFIG.shadow` (strength, maxLen-клампы, softness) — крутятся
без пересборки.

## 6. Сглаживание

`F`, `H_person` сглаживаются One-Euro/экспонентой (как `SmoothedShadow` сейчас),
чтобы дрожание bbox/дистанции не дёргало угол. Потеря фигуры → плавное затухание.

## 7. Ошибки / деградация

| Сбой | Поведение |
|---|---|
| нет `lights.json` у мира | тень-фолбэк: текущая «вниз» (uDrop), без падения |
| телеметрия без bbox/distance | фиксированная точка стояния (центр лаунжа) |
| `H_person` шумит | клампим + сглаживаем; дефолт 1.7 |
| лампа ниже роста (`z ≤ H`) | пропускаем эту лампу (нефиз. длина) |

## 8. Тестирование (TDD, hard rules — приоритет)

Юниты (`shadowGeom.test.ts`):
- `floorShadow`: верхняя лампа (z=3, d мал) → короткая тень; боковая низкая →
  длинная; направление строго от лампы; сдвиг `F` западнее → азимут меняется
  в верную сторону; лампа `z≤H` → пропуск.
- `personFloorWorld`: дальше дистанция → `F` дальше по оси взгляда; смещение
  bbox влево → `F` смещается верно; рост из bbox+дистанции в диапазоне.
- `projectFloor`: известная мировая точка пола → известный экранный пиксель
  (сверка с Blender-рендером прокси).
Шейдер/много-ламп — визуальная приёмка против Blender-прокси «Human 2»
(эталон): направление/длина/мягкость совпадают.

## 9. Этапы (декомпозиция)

1. Экспорт `lights.json` (лампы+камера) из Blender.
2. `shadowGeom.ts` + юниты (ядро, без рендера).
3. Компоновщик: ОДНА лампа (ключ) — проверить на живом, что ложится как у прокси.
4. 2-3 лампы + сглаживание + penumbra-софт.
5. Приёмка против прокси, тюнинг `LUX_CONFIG.shadow`.

## 10. Критерии приёмки

1. Сдвиг посетителя вбок/по глубине меняет направление и длину тени физично.
2. Поза (рука/наклон/нога) отражается в тени.
3. Направление/длина/мягкость совпадают с Blender-рендером прокси в той же точке.
4. Нет дрожи; потеря фигуры → плавное затухание; без `lights.json` — фолбэк.
5. Веб-юниты зелёные (существующие 87 не сломаны); рендер ≥ текущего fps.
