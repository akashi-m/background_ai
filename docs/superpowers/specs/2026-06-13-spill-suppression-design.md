# Спилл: чистый цвет края из RVM fgr — спека

**Дата:** 2026-06-13
**Статус:** утверждён заказчиком в чате («го спилл»)
**Родитель:** `2026-06-13-matte-quality-design.md` (A/B показал светлый ореол
по верху волос у обеих моделей — просочившийся фон в полупрозрачном крае)

## 1. Открытие

RVM предсказывает ДВА выхода: `fgr` (деконтаминированный цвет переднего плана —
модель сама вычищает подмешанный фон из края) и `pha` (альфа). Сейчас
`RvmEngine.process` возвращает только `pha`, а `fgr` выбрасывается; в SBS
пакуется сырой кадр камеры — вместе с ореолом. Официальная композиция авторов
RVM: `fgr * pha + bg * (1 - pha)`.

## 2. Решение

Контракт движка меняется: `process(rgb) → (fg_rgb, alpha)`:
- **RvmEngine** возвращает `fgr` (float [1,3,H,W] → uint8 [H,W,3]) — ноль
  дополнительных вычислений, модель уже посчитала;
- **MediapipeEngine** возвращает входной rgb как есть (деконтаминации нет);
- **Pipeline** пакует в SBS цвет от движка, не сырой кадр.

Контракт SBS с рендерером не меняется (RGB|A, та же форма) — рендерер
не трогаем, спилл уходит «в источнике».

## 3. Риск

`fgr` внутри фигуры (alpha=1) — реконструкция модели, теоретически чуть мягче
фактуры исходника. Проверка глазами по `/frame.png` после внедрения; если
заметно — смешивание fgr только в зоне края (alpha<0.99) отдельным шагом.

## 4. Тестирование

пайплайн пакует цвет движка (фейк красит RGB → левая половина SBS = окрашенный).
