**Дата:** 2026-06-19
**Ветка:** `lux/renderer`
**Точка отката:** git-тег `pre-stage-restructure` (HEAD 72eecc5)
**Статус:** дизайн на ревью

## 1. Контекст и цель

AR «волшебное зеркало» (Stellar Mirror): живой человек снимается камерой, матируется и
композитится в 3D-интерьер на вертикальном киоск-экране, ведя себя как зеркало.
Два процесса: Python-капчур (`capture/src/capture/`) и TS-рендерер (`src/`), связанные
WebRTC SBS-видео (левая половина RGB, правая — альфа) + WS-телеметрия JSON.

Сегодня композит — это **один процедурный метод** `LuxCompositor.render()`
([compositor.ts:556-795](../../../src/lux/compositor.ts)) и **один мегашейдер** `personMat`
([compositor.ts:273-335](../../../src/lux/compositor.ts)), фьюзящий 7 операций. Все «look»-ручки
в **глобальном** `LUX_CONFIG` ([config.ts](../../../src/lux/config.ts)); пер-интерьерны только
LUT, `lightDirX`, `shadowStrength`, тени-данные (в `meta.json`).

**Цель:** разложить композит на явные, именованные, тюнящиеся и пер-интерьерные стадии;
добавить две отсутствующие стадии (общий фильтр кадра и привязку роста 1:1); вынести
конфиги по каждому интерьеру в отдельные файлы, редактируемые dev-панелью с записью на диск.

### 1.1 Северная звезда и зафиксированные решения

- **D1 — Максимальный фотореализм** — единственное жёсткое требование. Железо выбирается
  под продукт (не наоборот). Перф/стоимость кадра вторичны: предпочитаем **чистое разделение
  стадий**, а не слипание ради fps.
- **D2 — Проектируем цельно сейчас**, реализуем фазами. Откат-тег `pre-stage-restructure` стоит.
- **D3 — Де-шэдоу/нормализация = стадия-слот.** Старт — «lite» (выровнять жёсткие тени/пересветы
  + цвет/экспозиция под фон). Контракт стадии обязан допускать будущий полный image-based
  **relight** без правок ниже по пайплайну.
- **D4 — Один посетитель сейчас**, но все структуры данных человека — **массивы, ключ по `id`**,
  чтобы N-человек добавились без переписывания.
- **D5 — Пер-интерьерный конфиг** — в `look.json` каждого мира (геометрия/контент остаются в
  `meta.json`). Dev-панель (`G`) правит активный мир вживую и **пишет `look.json` на диск** через
  мелкий Vite dev-эндпоинт. Глобальные дефолты мёржатся под пер-интерьерные оверрайды.
- **D6 — Камера: ZED (глубинная) как основа + монокуляр-вебка как полноценный fallback.**
  ZED-зависимый реализм — отдельным hardware-треком позади низкорискового костяка.
- **D7 — Зеркало = life-size постоянный 1:1.** Аватар всегда в натуральный рост и **не сжимается**
  при отходе (это желаемое поведение киоска, а не оптически-честное отражение).

### 1.2 Главное архитектурное решение

**Эволюция в `Stage[]`, а НЕ декларативный framegraph.**
Порядок проходов и топология FBO в `compositor.ts` **уже корректны** — выигрыш в том, чтобы
сделать их читаемыми и тюнящимися, а не валидировать в рантайме. Framegraph закрывал бы ровно
один латентный хазард (переиспользование `wrapRT_A` под bloom), который мы чиним бесплатно
(гейт bloom по `mirrorVisible`). Рантайм-перестановка стадий из JSON на живом киоске —
угроза «окирпичивания» экрана; вместо неё — `enabled` + пер-стадийные `params` (тоггл и тюн),
порядок фиксирован в коде. Это сохраняет границу `compositor.render(opts)` и существующие
юнит-тесты.

Отвергнутые альтернативы (для возможного override владельцем продукта):
1. **Декларативный framegraph** — рантайм-перестановка = liveness-риск; польза графа (ловля
   RT-хазардов) закрывает один уже-известный баг. Взяли только его дизайн `look.json`.
   *Override:* если нужны реально data-driven пер-интерьерные **переставляемые** пайплайны.
2. **Реализм-максималист «всё на ZED»** — высший потолок реализма, но завязан на железо,
   которого нет (`zed.py` — заглушка), и без него тихо схлопывается. Взяли его идеи (общий
   фильтр, слот relight, разведение трёх FOV) как **отдельный hw-трек позади костяка**.

## 2. Целевой конвейер: 11 явных стадий

`(PY)` = Python-капчур · `(GPU)` = проход рендера · `(CPU)` = TS-математика, кормящая GPU.
«look» = параметр тюнится на интерьер в `look.json`.

| # | Стадия | Где | Пер-интерьерные параметры |
|---|--------|-----|---------------------------|
| **S1 CameraFrame** | декод источника (webcam/file/ZED) → `frame.rgb (+depth?)` | PY | нет (rig-уровень, `capture/config.py`) |
| **S2 ExtractPeople** | RVM resnet50 матт, recurrent-reset на вход → SBS + `people[]` телеметрия | PY (матт) + GPU (только SBS-распаковка/флип) | `matte.erode`, `matte.feather[lo,hi]` (опц.) |
| **S3 DeShadow+Normalize** *(СЛОТ, lite сейчас)* | локальная нормализация **собственного** света человека (выровнять жёсткие тени/пересветы к нейтрали) → `personA`. Scene-относительный цвет — НЕ здесь, а в S9 | GPU (НОВАЯ; выделяется из `personMat`) | `deshadow{mode,strength,localLumaRadius,highlightKnee,shadowLift}` |
| **S4 CropBadEdges** | затухание у кромки кадра + гард SBS-шва по альфе | GPU (тонкий проход по альфе `personA`) | `crop{edgeFade,sbsSeamGuard}` |
| **S5 SceneBackground** | 3D-мир ИЛИ cover-fit плейт → `sceneRT` | GPU | нет (геометрия/контент в `meta.json`) |
| **S6 SceneStats** | light-wrap даунскейл + 2-pass блюр → `wrapA`; 8×8 редукция → `meanRT` 1×1 | GPU | `wrapBlurPx` |
| **S7 PlacePerson (геометрия зеркала)** | **привязка роста 1:1** + якорь к полу; блит фигуры в `compositeRT`. Даёт **общий** масштаб фигуры+тени (см. §4) | CPU (рост) + GPU (блит) | `geom{anchorMode,mirrorMag,photo25dHeightFudge,footAnchorRaise,heightTrust}` |
| **S8 CastShadow** | запечённая база + прокси-мультиплай + контактный блоб; фолбэк-силуэт | CPU (`ProxyRig`) + GPU (мульти-проход) | **весь** блок `shadow{}` (крупнейший выигрыш конфига) |
| **S9 MatchGrade** | сатурация/контраст/температура/экспозиция фигуры под `meanRT` | GPU (в зоне person-прохода) | `grade{colorMatch{cast,exposure},contrast,temp,saturation,shade,lightDirX}` |
| **S10 UnifyFrame** | **на весь кадр** 3D-LUT + глобальный грейд + зерно + блум по `compositeRT` (комната+человек+тень вместе) → экран | GPU (НОВАЯ + существующие bloom/grain) | `unify{lut,lutStrength,grain,bloom,bloomThreshold,vignette}` |
| **S11 Transition** | кроссфейд слайдшоу + шторка смены миров | GPU | нет (UX-тайминги в `LUX_CONFIG`) |

### 2.1 Что выделяем из `personMat`, что оставляем слитным

Судьи подтвердили: текущая фьюзия в `personMat` ([compositor.ts:273-335](../../../src/lux/compositor.ts))
во многом **корректна** — alpha-клампы половины SBS (`ax0/ax1`, строки 283-289) обязаны идти
**до** feather/edge-fade, т.к. делят `uvm`.

- **Выделяем** (повышает реализм или даёт реальный шов плагина): **LUT** → S10 (на весь кадр),
  **де-шэдоу/нормализация** → S3 (слот D3), **обрезка краёв** → S4.
- **Оставляем слитным в person-проходе:** SBS-распаковка + флип + эрозия альфы + feather +
  light-wrap + математика colorMatch (S9). Это тюнится юниформами пер-интерьер, но **физически
  не переставляется**.

### 2.2 Концептуальные стадии ≠ физические проходы

Таблица §2 — **концептуальные** единицы (как думает автор/артист). Физически часть из них
объединена в один **person-проход** (эволюция `personMat`): он рисуется **после** S8 и несёт
SBS-распаковку (часть S2) + ядро (эрозия/feather/light-wrap) + грейд (S9). Отдельными проходами
идут только pre-обработка (**S3** де-шэдоу, **S4** обрезка краёв) и post-обработка (**S10** общий
фильтр). **S7 — это CPU-вычисление** общего масштаба+якоря (а не отдельный draw): результат кормит
И тени S8, И person-проход; сами пиксели фигуры рисуются в person-проходе.

**Порядок выполнения рантайма** (важно — отличается от концептуальной нумерации):
S1→S2 → **S5→S6** (сцена+статы первыми, чтобы `meanRT`/`wrapA` были готовы) → **S3→S4**
(pre-обработка человека в `personA`) → **S7** (масштаб+якорь) → **S8** (тени) →
**person-проход** (place + ядро + S9-grade) → **S10** → **S11**.

## 3. Пер-интерьерный конфиг — `look.json` (D5)

Файл `public/assets/worlds/<name>/look.json`. **`meta.json` хранит геометрию/контент**
(format, file, depthFile, aspect, transform, dollyMaxCm, depthAmountCm, flat, lightDirX,
shadow.lightsFile/worldPosFile). **Грейд + тон тени переезжают в `look.json`.** Все ключи
опциональны; делается **глубокий мёрж по листам** (мир побеждает) над глобальными дефолтами.

```jsonc
{
  "matte":   { "erode": 0.0025, "feather": [0.4, 0.8] },
  "deshadow":{ "mode": "lite", "strength": 0.5, "localLumaRadius": 0.04,
               "highlightKnee": 0.8, "shadowLift": 0.1, "lampsetRef": null },
  "crop":    { "edgeFade": 0.03, "sbsSeamGuard": 0.0015 },
  "geom":    { "anchorMode": "coverfit", "mirrorMag": 1.0,
               "photo25dHeightFudge": 1.0, "footAnchorRaise": 0.04, "heightTrust": 1.0 },
  "grade":   { "colorMatch": { "cast": 0.35, "exposure": 0.15 },
               "contrast": 1.08, "temp": 0.02, "saturation": 1.0, "shade": 0.18 },
  "shadow":  {
    "strength": 0.5,
    "baked":  { "feetUV": [0.233, 0.161], "raise": 0.05 },
    "proxy":  { "centerDark": 0.12, "edgeDark": 0.03, "blur": 0.014, "shrinkK": 0.89, "feetCutR": 0.16 },
    "blob":   { "opacity": 0.36, "raise": 0.04, "ratioY": 0.3 },
    "multiply": { "tint": [0.40, 0.35, 0.28], "centerDark": 0.36, "edgeDark": 0.072,
                  "blur": 0.009, "maxShadow": 0.5 },
    "softness": 1.6, "bias": 0.005
  },
  "unify":   { "lut": "interior.cube", "lutStrength": 1.0, "grain": 0.07,
               "bloom": 0.5, "bloomThreshold": 0.72, "vignette": 0.0 }
}
```

### 3.1 Критические правки схемы (НЕ как в черновиках)

- **Тон тени разнесён по проходам.** `shadow.multiply.centerDark = 0.36` (дефолт материала
  `multiplyBlit`) — это **другое поле**, чем `shadow.proxy.centerDark = 0.12` (рантайм-оверрайд
  [compositor.ts:678](../../../src/lux/compositor.ts)). Слияние их в одно поле **молча утроило бы**
  тьму прокси при миграции.
- **`shrinkK = 0.89` — независимый эстетический параметр прокси-тени**, НЕ артефакт роста: прокси
  уже использует реальный `H` ([shadowScene3D.ts:123](../../../src/lux/shadowScene3D.ts)),
  инвариант `z/y=0.8` закреплён тестом. Привязка роста 1:1 его **не отменяет**.
- **`bloomThreshold` живой** (роутится в [compositor.ts:504](../../../src/lux/compositor.ts)) —
  пробел только в персистентности.
- **Единая сила тени.** Сегодня пер-интерьерный `shadowStrength` молча множится на захардкоженный
  `multiplyBlitMat.uShadowStrength = 0.5` (0.8 → эффективно 0.4). Схлопываем в один `shadow.strength`.

### 3.2 Мёрж и гарантия безопасности

`resolveLook(global, world)` — глубокий мёрж по листам, мир побеждает. `LUX_CONFIG`
реструктурируется в **точно эту форму** как глобальные дефолты, значения **байт-в-байт равны
сегодняшним константам**. Отсутствует `look.json` ⇒ чистые дефолты ⇒ **пиксельно идентично
сегодняшнему кадру**. Это главная страховка миграции — закрепляется deep-equal snapshot-тестом.

### 3.3 Dev-панель → диск (D5)

`vite.config.ts` сегодня отсутствует (проект на дефолтах Vite). Добавляем минимальный:

- Плагин `lookWriterPlugin()`, **`apply: 'serve'`** (никогда в build), эндпоинт `POST /__look/:world`.
  Пишет тело JSON в `public/assets/worlds/:world/look.json`, **имя мира санитизируется по
  allowlist живых миров** (серверный гард от path-traversal).
- `devPanel.ts` (`G`) получает кнопку **«Save to look.json»** рядом с Copy. Слайдеры уже роутятся
  вживую через `compositor.setTuning`; расширяем его на новые look-ключи (тон теней по проходам,
  `deshadow.strength`, `crop.edgeFade`, `unify.lut`). Save сериализует текущие resolved-значения
  активного мира и POST'ит.
- **Привязка к активному миру:** панель строится с геттером `() => worlds[switcher.index].name`,
  Save всегда целится в видимый интерьер; при смене мира слайдеры пере-сидятся из его resolved-look.
  Слайдеры генерируются из схемы look, новый параметр авто-создаёт слайдер.

## 4. Геометрия зеркала — привязка роста 1:1 (S7)

Сегодня фигура **cover-fit** ([compositor.ts:613-620](../../../src/lux/compositor.ts)), а `H`
([shadowGeom.ts:78](../../../src/lux/shadowGeom.ts), Blender-bake `fovY`, кламп [1.4,2.0], без
гистерезиса) кормит **только** прокси-тень. Это буквально отсутствующая стадия.

### 4.1 Разведение трёх FOV — по одной роли на источник

- **FOV экран-калибровки** ([offAxis.ts](../../../src/render/offAxis.ts), `screenWcm/Hcm` + eye)
  → **только** off-axis фрустум фона/параллакс. Единственный источник «world-cm на screen-cm в
  плоскости зеркала».
- **Blender-bake `fovY`** ([shadowGeom.ts:9-14](../../../src/lux/shadowGeom.ts)) → **только**
  размещение тени в комнате (`personFloorWorld`, `ShadowScene3D`). **Не** размеряет фигуру.
- **`k=110`** (`presence.py`) → понижен до **хинта присутствия/дистанции**, заменяется метрикой
  ZED, когда доступна.

### 4.2 Математика привязки роста (новый `src/lux/mirrorGeom.ts`, чистый + юнит-тест)

1. Реальный рост `H_m` (м): **приоритет — метрика ZED** (макушка минус пол в облаке точек).
   Fallback: через **capture-FOV** (не Blender), `d = distanceCm/100`,
   `frameH = 2·tan(captureVfov/2)·d`, `H_m = bboxHeightRatio · frameH`. Прогон через `heightTrust`
   (1.0 = метрика; <1 лерпит к фикс. 170 см, гася монокуляр-дрожь). **Добавить EWMA + гистерезис
   вход/выход** по `H_m` (сегодня их нет — подтверждённый источник дрожи).
2. Физический масштаб экрана: `pxPerCm = screen.heightPx / calibration.screenHcm`.
   Целевая высота фигуры `H_px = H_m · 100 · pxPerCm · mirrorMag`.
3. UV-масштаб: `sy = (H_px / canvasHeightPx) / personAlphaBboxH`;
   `sx = sy · personAspect / canvasAspect` (без растяжения). Заменяет наивный cover-fit `sx/sy`.
4. Якорь: `uUvOffset` так, чтобы `feetUV` ([main.ts:206](../../../src/main.ts)) лёг на линию
   контакта с полом на экране.

### 4.3 Фигура и тень — ОДИН масштаб (правка, которую требовали судьи)

Комментарий [compositor.ts:613](../../../src/lux/compositor.ts) — *«общий масштаб для тени и
фигуры»* — и тот же `sx/sy` кормит `groundShadowMat` и `blobMat`. Поэтому **S7 выдаёт ОДИН
согласованный масштаб, который потребляют И блит фигуры, И контактные тени (S8)**. Привязка
роста — **не** «только фигура» (это был баг черновика: тень бы «отклеилась» от ног).
`personAlphaBboxH` TS-side сегодня не считается → S7 берёт его из телеметрии `people[i].bbox`.

### 4.4 Честность `mirrorMag` и flat vs photo25d

- **`mirrorMag` (дефолт 1.0)** — отображение реального роста в экранный **независимо от дистанции**
  = life-size проекция (D7), не оптическое зеркало. Для киоска это и нужно; документируем как
  «life-size mirror», не заявляем оптическую корректность.
- **Только `flat:true` миры — истинные зеркала** (без depth-смещения). photo25d-миры получают
  привязку роста с пер-интерьерным `photo25dHeightFudge` (дефолт 1.0) и флаг `mirrorFidelity:'approx'`.
  **Рекомендация: критичные к зеркалу интерьеры авторить как flat-плейты.**
- `living` — `flat:true` И `format:photo25d` одновременно (это **не** взаимоисключение):
  `flat` = переключатель зеркало-точности, `format` = тип ассета. `flat` (в `meta.json`) остаётся
  единственным источником решения о зеркале; дубля `geom.depthMode` не вводим.

## 5. Слот де-шэдоу (D3)

S3 — первоклассная `Stage` с замороженным контрактом: **lite сейчас, relight потом без правок ниже**.

```ts
export interface DeShadowStage extends Stage {
  // читает: сырой RGB человека (SBS) + meanRT (+ опц. lampset/нормали интерьера)
  // пишет:  ctx.rt.personA  (RGB трансформирован; АЛЬФА НЕ ТРОГАЕТСЯ — инвариант графа)
  // mode из ctx.look.deshadow.mode
}
```

- **Lite** (`deshadowStage.ts`, первый): локальная яркость через малый сепарабельный блюр
  (переиспользуем `blurMat` в `personB`), нормализация **собственного** света человека к целевому
  миду (`strength` лерп, `highlightKnee` срезает выбитую кожу/одежду, `shadowLift` открывает тени).
  **Self-contained — без сцены/`meanRT`** (scene-относительный цвет/экспозиция — задача S9, где
  `meanRT` уже готов; статичные лампы для будущего relight грузятся из `shadowData`, тоже не зависят
  от порядка).
- **Relight** (будущий `relightStage.ts`): тот же контракт, `mode:"relight"`, дополнительно берёт
  лампы интерьера (`shadowData.lamps`, [worldScene.ts:118](../../../src/scenes/worldScene.ts)) +
  пер-пиксельные нормали (depth ZED или псевдо-нормали из матта как graceful fallback),
  пересвечивает под рig комнаты. Подмена = одна строка в `buildGraph` по `look.deshadow.mode`;
  S4-S10 знают только `personA`.

Инвариант, делающий подмену безопасной: **де-шэдоу не трогает геометрию (S7) и финальный
грейд (S10) — только RGB человека по контексту освещения.**

## 6. Готовность к N-человек (D4)

Ключуем всё по `id` от стадии матта; N=1 сейчас, без переписывания под N>1:

- **Капчур:** `PipelineStats` несёт `people: list[PersonState]`, где
  `PersonState{id, present, distanceCm, heightCm, bbox, pose{world,norm,healthy}}`. `server.py`
  шлёт `{"type":"presence","people":[...]}`. **Single-person поля зеркалятся в `people[0]`** на
  переходный период, чтобы устаревший рендерер не ломал WS-контракт. RVM recurrent-reset —
  пер-поток сейчас, пер-трек потом.
- **Рендерер:** `StageCtx.people: PersonState[]`. `smoothF/smoothH/poseSmoother/feetUV` в `main.ts`
  → `Map<id, …>`. Каждая person-стадия итерирует `for (const p of ctx.people)`; кадровые стадии
  (S5,S6,S10,S11) count-agnostic. `personA/personB` → 2×2 атлас RT для N≤4.
- **Честно о границе:** форма данных бесплатна, но **истинный** мульти-человек требует
  instance-сегментации в Python-матте + снятия `num_poses=1` — это реальная капчур-работа,
  отложена. Архитектура готова; матт — настоящий объём.

## 7. Общий фильтр на весь кадр (S10, цель-стадия 8)

Корень («чтобы всё слилось в одну текстуру»): 3D-LUT применяется **только внутри `personMat`**
([compositor.ts:299-303](../../../src/lux/compositor.ts)) → фон сквозь него не проходит, фигура и
комната — две текстуры. Чиним через **S10 UnifyFrame**: после S7 (постановка) + S8 (тень) +
S9 (грейд) записали комнату+человека+тень в `compositeRT`, делаем **один проход на весь кадр**:
LUT интерьера + глобальный контраст/температура/сатурация, затем зерно + аддитивный блум —
**всё по всему кадру**. Пер-person LUT по умолчанию **выключаем** (`uLutOn → 0`); цвет-идентичность
применяется один раз ко всему. Общее зерно + общий блум + один LUT + один грейд над обоими слоями =
буквальная гарантия «одной текстуры». `unify.lutStrength` — интенсивность; `unify.lutOn` пер-мир —
opt-out для плейта с уже-запечённым LUT. Блум гейтится по `mirrorVisible` (заодно убивает латентный
хазард `wrapRT_A`).

## 8. Поэтапный rollout

Каждая фаза — вертикальный срез; зеркало остаётся рабочим; откат безопасен (тег `pre-stage-restructure`).

- **P0 — Сейф-нет (без смены поведения).** Добавить **отсутствующие** golden-image readback +
  stage-order тесты (сегодня их нет). Снимок кадра «living»; ассерт порядка стадий. *Проверка:*
  pixel-diff ≈ 0. Детерминированный headless-WebGL readback нетривиален (зерно/время) — заморозить
  seed зерна и часы в тест-режиме.
- **P1 — Подъём look-конфига (без смены поведения).** `src/lux/look.ts` + `ResolvedLook`;
  реструктура `LUX_CONFIG` в форму look с **байт-равными дефолтами**. Заменить литералы шейдеров и
  модульные const на юниформы из `resolveLook`. **Разнести тон тени по проходам** и схлопнуть
  двойной множитель `shadowStrength × 0.5` в один `shadow.strength`. Переезд `lut` из `meta.json`
  (и его парсера/теста в `worldMeta.ts`) в `look.json`. *Проверка:* golden неизменен И
  `resolveLook(defaults)` deep-equal замороженному снимку сегодняшних констант.
- **P2 — Обёртка стадий (без смены поведения).** `render()` → `buildPrep → цикл Stage[] → present`,
  стадии оборачивают ТЕ ЖЕ проходы в ТОМ ЖЕ порядке. Добавить `personA/personB` RT (пока не исп.).
  Гейт bloom по `mirrorVisible` (убивает хазард `wrapRT_A`). Удалить мёртвый `if(false)` roomShadowMat
  ([compositor.ts:694](../../../src/lux/compositor.ts)). *Проверка:* golden неизменен; порядок-тест ок.
- **P3 — Пер-мир `look.json` + dev-save (opt-in, D5).** `vite.config.ts` + `lookWriterPlugin`
  (serve-only, allowlist), кнопка Save + геттер активного мира, мёрж `resolveLook`. Авторить
  identity `look.json` для `living`, затем тюнить тон тени пер-интерьер. *Проверка:* мир без
  look.json == дефолты; identity look.json == golden; Save round-trip на диск; эндпоинт отвергает
  неизвестные имена миров.
- **P4 — Unify-LUT на весь кадр (новая стадия, намеренная смена).** Добавить S10; перенести LUT с
  человека на весь кадр; пер-мир `unify.lut`; person-LUT по умолчанию off. *Проверка:* golden
  обновлён **намеренно**; тест сэмплит пиксель фона и пиксель человека через один LUT.
- **P5 — Геометрия зеркала (намеренная смена, под флагом).** `mirrorGeom.ts` привязка роста вместо
  cover-fit, под `geom.anchorMode` (legacy `coverfit` по умолчанию ВКЛ до валидации), **кормит общий
  масштаб фигуры+тени**. Добавить гистерезис H. *Проверка:* на flat-мире тестер 1.8 м читается ~1.8 м
  на экране И контактная тень держится у ног; откат = `anchorMode:'coverfit'`.
- **P6 — Де-шэдоу lite (новая стадия, по умолчанию off).** `deshadowStage` → `personA`; person-проход
  читает `personA` при `deshadow.mode != 'off'`. Шип default-off, включать пер-интерьер после A/B.
  *Проверка:* off == golden; on = ровная кожа под жёстким светом, без артефактов края (следить за
  premult/alpha-clamp дрейфом).
- **P7 — Метрика капчура + форма данных N-человек (D4 + hw-трек D6).** `people[]` телеметрия
  (N=1, шим `people[0]`), ZED-источник кормит S7 метрический рост + S3 relight-нормали. *Проверка:*
  тесты парсинга телеметрии; вебка-fallback неизменна; ZED `heightCm` совпадает с рулеткой в допуске.
- **P8 — Drop-in relight (будущее, без правок пайплайна).** `relightStage` под `deshadow.mode:'relight'`;
  новый файл + свитч. *Проверка:* существующие режимы байт-идентичны; relight чисто аддитивен.

## 9. Риски

1. **Golden-харнесс — линчпин** «без смены поведения» для P0-P5, его сегодня нет; детерминированный
   WebGL-readback сложнее, чем кажется. Слабый допуск → каждая «сохраняет пиксели» фаза тихо дрейфует.
   *Это гейт — фондируем первым.*
2. **Дефолты `look.json` обязаны == сегодняшним константам точно**, с разнесённым по проходам тоном
   тени, иначе миграция молча пере-грейдит каждый интерьер (ловушки `shadowStrength → 0.4` и `centerDark` ×3).
   Митигируется deep-equal snapshot-тестом.
3. **Привязка роста меняет размер фигуры на каждом существующем демо** — самый заметный регресс.
   Под флагом, flat-миры первыми, нужна приёмка на месте с тестером известного роста.
4. **Сплит мегашейдера** (де-шэдоу, LUT) рискует premult-alpha / alpha-clamp дрейфом края; golden-тест
   обязан ловить.

## 10. Рекомендация по железу (D1)

- **Камера: ZED 2i (стерео-глубина).** Keystone для D1: честная метрическая дистанция/рост (убирает
  шумный `k=110` и Blender-`fovY` хак), включает истинную привязку 1:1, даёт пер-пиксельные
  нормали/глубину под image-based relight. Провод через `sources/zed.py` (сегодня заглушка);
  нужен Windows/RTX-хост + ZED SDK + `pyzed`. Вебка-монокуляр — первоклассный graceful fallback;
  стадии **никогда** не завязывать жёстко на наличие глубины.
- **Свет: контролируемый ровный мягкий фронтальный fill на зоне посетителя.** Два больших
  диффузных софтбокса/LED ~45° по бокам киоска + низкий ambient-баунс от пола — сгладить жёсткие
  само-тени и пересветы **на съёмке**. Чем чище свет захвата, тем меньше работы у lite-де-шэдоу и
  тем убедительнее relight. CRI >95 для верных тонов кожи через LUT.
- **Экран: вертикальный full-height киоск-панель в нативном 9:16 (aspect 0.5625), цвет-калиброванный.**
  Физ. высота ≈ реального стоящего человека (видимые ≥ ~190-200 см), чтобы life-size читался верно;
  точно калибровать `screenHcm/screenWcm` (исп. offAxis + привязка роста) на установке. Матовая
  антибликовая поверхность, широкий гамут, заводская/по-юнитная калибровка — чтобы пер-интерьерный
  LUT ложился одинаково на каждом киоске.
